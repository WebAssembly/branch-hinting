Binary Format¶

Code Metadata¶

All code metadata items of a given type T are grouped under a custom section named ‘metadata.code.T’. The following parametrized grammar rules define the generic structure of a code metadata section of type T.

\[\begin{split}\begin{array}{llcll} \def\mathdef1#1{{}}\mathdef1{code metadata section} & \href{../appendix/custom.html#binary-branchhintsec}{\mathtt{codemetadatasec}}(\mathtt{T}) &::=& \href{../binary/modules.html#binary-section}{\mathtt{section}}_0(\href{../appendix/custom.html#binary-branchhintsec}{\mathtt{codemetadata}}(\mathtt{T})) \\ \def\mathdef1#1{{}}\mathdef1{code metadata} & \href{../appendix/custom.html#binary-branchhintsec}{\mathtt{codemetadata}}(\mathtt{T}) &::=& n{:}\href{../binary/values.html#binary-name}{\mathtt{name}} & (\mathrel{\mbox{if}} n = \def\mathdef40#1{\mbox{‘}\mathtt{#1}\mbox{’}}\mathdef40{metadata.code.T}) \\ &&& \href{../binary/conventions.html#binary-vec}{\mathtt{vec}}(\href{../appendix/custom.html#binary-branchhintssec}{\mathtt{codemetadatafunc}}(\mathtt{T})) \\ \def\mathdef1#1{{}}\mathdef1{code metadata function} & \href{../appendix/custom.html#binary-branchhintssec}{\mathtt{codemetadatafunc}}(\mathtt{T}) &::=& fidx{:}\href{../binary/modules.html#binary-funcidx}{\mathtt{funcidx}}~\href{../binary/conventions.html#binary-vec}{\mathtt{vec}}(\href{../appendix/custom.html#binary-branchhintsec}{\mathtt{codemetadataitem}}(\mathtt{T})) \\ \def\mathdef1#1{{}}\mathdef1{code metadata item} & \href{../appendix/custom.html#binary-branchhintsec}{\mathtt{codemetadataitem}}(\mathtt{T}) &::=& \mathit{instoff}{:}\href{../binary/values.html#binary-int}{\def\mathdef6#1{{\mathtt{u}#1}}\mathdef6{\mathtt{32}}} ~~ \mathit{size}{:}\href{../binary/values.html#binary-int}{\def\mathdef6#1{{\mathtt{u}#1}}\mathdef6{\mathtt{32}}} & (\mathrel{\mbox{if}} \mathit{size} = ||\mathtt{T}||) \\ &&& \mathit{data}{:}\mathtt{T} \\ \end{array}\end{split}\]

Where funcpos is the byte offset of the annotation starting from the beginning of the function body, and data is a further payload, whose content depends on the type T.

code metadata function entries must appear in order of increasing function id, and duplicate id values are not allowed. code metadata item entries must appear in order of increasing instruction offset, and duplicate offset values are not allowed.

Branch Hints¶

A Branch Hint is code metadata item with type branch_hint. All branch hints for a module are contained in a single code metadata section with name ‘metadata.code.branch_hint’.

\[\begin{split}\begin{array}{llcll} \def\mathdef1#1{{}}\mathdef1{branch hint section} & \href{../appendix/custom.html#binary-branchhintsec}{\mathtt{branchhintsec}} &::=& \href{../appendix/custom.html#binary-branchhintsec}{\mathtt{codemetadatasec}}(\href{../appendix/custom.html#binary-branchhintsec}{\mathtt{branch\_hint}}) \\ \def\mathdef1#1{{}}\mathdef1{branch hint} & \href{../appendix/custom.html#binary-branchhintsec}{\mathtt{branch\_hint}} &::=& \def\mathdef41#1{\mathtt{0x#1}}\mathdef41{00} \\ &&|& \def\mathdef42#1{\mathtt{0x#1}}\mathdef42{01} \\ \end{array}\end{split}\]
